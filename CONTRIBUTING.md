# Contributing to AKP

## Workflow

1. Add or update the source entity in `entities/`.
2. Confirm the entity exists in `registry/akp-registry.csv`.
3. Use YAML front matter for every entity file.
4. Keep language educational, clear, and compliance-conscious.
5. Generate publishing outputs only after the source entity is complete.

## Naming Conventions

Use lowercase, hyphenated filenames.

## Entity IDs

- `COMP-001`
- `PROG-001`
- `PERSONA-001`
- `PROC-001`
- `FAQ-0001`
- `CALC-001`
